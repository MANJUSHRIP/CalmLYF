================================================================================
                          JAVASCRIPT FUNCTIONS
================================================================================

1. selectUserType(type)
   - Purpose: Handle user type selection (client or doctor)
   - Parameters: type = "client" or "doctor"
   - Actions: Store type, hide selection, show forms

2. backToUserType()
   - Purpose: Return to user type selection
   - Parameters: None
   - Actions: Reset forms, show selection screen

3. switchAuthTab(tab)
   - Purpose: Switch between login and signup tabs
   - Parameters: tab = "login" or "signup"
   - Actions: Show/hide forms, update tab styling

4. handleLogin(event)
   - Purpose: Process login form submission
   - Parameters: event = form submit event
   - Actions: Validate, store data, close modal

5. handleSignup(event)
   - Purpose: Process signup form submission
   - Parameters: event = form submit event
   - Actions: Validate, check passwords, store data, close modal

6. continueAsGuest()
   - Purpose: Allow access without account
   - Parameters: None
   - Actions: Set guest mode, close modal

7. closeAuthModal()
   - Purpose: Close auth modal and show website
   - Parameters: None
   - Actions: Hide modal, show navbar/sections, navigate to home

8. logout() [Optional - can be added]
   - Purpose: Logout user
   - Parameters: None
   - Actions: Clear localStorage, reload page

================================================================================
                          DATA STORAGE
================================================================================

localStorage Keys:

1. isLoggedIn
   - Type: String
   - Values: "true" or "false"
   - Purpose: Track if user is logged in
   - Example: "true"

2. userType
   - Type: String
   - Values: "client", "doctor", or "guest"
   - Purpose: Store user's role
   - Example: "client"

3. userData
   - Type: JSON String
   - Purpose: Store user information
   - Example: {
       "email": "user@example.com",
       "userType": "client",
       "loginTime": "2024-12-06T10:30:00Z",
       "name": "John Doe"
     }
